# Agent Loop 流程图

## 流程概述

```
┌─────────────────────────────────────────────────────────────────┐
│                        Agent Loop 流程                          │
└─────────────────────────────────────────────────────────────────┘
          │
          ▼
  用户输入: "你好，列出当前目录的文件"
          │
          ▼
┌─────────────────────────────────────────────────────────────────┐
│  第 1 次请求 (POST /v1/chat/completions)                        │
│  ─────────────────────────────────────────                      │
│  messages: [                                                    │
│    { role: "system", content: "You are a coding agent..." },    │
│    { role: "user", content: "你好，列出当前目录的文件" }          │
│  ]                                                              │
└─────────────────────────────────────────────────────────────────┘
          │
          ▼
┌─────────────────────────────────────────────────────────────────┐
│  LLM 处理 (qwen_qwen3.5-2b)                                    │
│  ─────────────────────────────────────────                      │
│  • prompt eval: 313 tokens, 545ms (574 tok/s)                   │
│  • eval: 23 tokens, 617ms (37 tok/s)                            │
│  • 生成 tool_call: [{ name: "bash", arguments: { command: "dir /B" } }] │
│  • finish_reason: "tool_calls"                                  │
└─────────────────────────────────────────────────────────────────┘
          │
          ▼
┌─────────────────────────────────────────────────────────────────┐
│  代码执行 run_bash("dir /B")                                    │
│  ─────────────────────────────────────────                      │
│  输出: .env, .env.example, .github, .gitignore, agents, ...     │
└─────────────────────────────────────────────────────────────────┘
          │
          ▼
┌─────────────────────────────────────────────────────────────────┐
│  第 2 次请求 (POST /v1/chat/completions)                        │
│  ─────────────────────────────────────────                      │
│  messages: [                                                    │
│    { role: "system", ... },                                     │
│    { role: "user", ... },                                       │
│    { role: "assistant", tool_calls: [{ name: "bash", arguments: { command: "dir /B" } }] },  ← 新增 │
│    { role: "tool", tool_call_id: "...", name: "bash", content: ".env\n.env.example\n.github\n.gitignore\nagents\n..." }        ← 新增 │
│  ]                                                              │
└─────────────────────────────────────────────────────────────────┘
          │
          ▼
┌─────────────────────────────────────────────────────────────────┐
│  LLM 处理                                                       │
│  ─────────────────────────────────────────                      │
│  • prompt eval: 100 tokens, 162ms (616 tok/s)                   │
│  • eval: 79 tokens, 1743ms (45 tok/s)                           │
│  • 生成文本回复 (无 tool_calls)                                  │
│  • finish_reason: "stop"                                        │
└─────────────────────────────────────────────────────────────────┘
          │
          ▼
  输出: "当前目录的文件如下：\n- .env\n- .env.example\n..."
          │
          ▼
    结束 (无 tool_calls，退出循环)
```

## 关键点

- 模型正确识别了 Windows 环境，使用了 `dir /B` 而非 `ls`
- 两次请求间缓存命中率高（checkpoint restore）
- 第二次请求后没有 tool_calls，循环结束
