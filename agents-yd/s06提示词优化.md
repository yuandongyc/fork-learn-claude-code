# s06 提示词优化思路

## 问题

运行 s06_context_compact_NV_R2.py 时，LLM 在 Windows 环境下选择了错误的 bash 命令 (`ls` 而非 `dir`)，导致工具调用失败。

**根因**：SYSTEM prompt 太简陋，没有告知 LLM 运行环境信息。

```python
SYSTEM = f"You are a coding agent at {WORKDIR}. Use tools to solve tasks."
```

## 解决思路

### 方案对比

| 方案 | 做法 | 适用场景 | 推荐度 |
|------|------|----------|--------|
| 1. 动态检测 | platform.system() 检测，告知 "Windows" | 工具少 | ⭐⭐ |
| 2. 明确约束 | 列出 `ls`→`dir`, `cat`→`type` 等映射 | 工具少 | ⭐⭐⭐ |
| 3. 简洁引导 | "优先用 read_file/glob" | 工具少 | ⭐⭐⭐⭐ |
| 4. 分层引导 + 工具自述 | SYSTEM 给出策略 + 工具 description 写环境适配 | 工具多 | ⭐⭐⭐⭐⭐ |

### 推荐：分层引导 + 工具自述

#### 1. SYSTEM prompt - 整体策略

```python
SYSTEM = (
    f"You are a coding agent at {WORKDIR}.\n"
    "Tool usage priority:\n"
    "1. read_file/glob - file reading & discovery (recommended)\n"
    "2. grep - content search in large codebases\n"
    "3. edit_file - text replacement\n"
    "4. bash - only when necessary\n"
    f"{'Windows environment: prefer read_file/glob over bash commands.' if IS_WINDOWS else ''}\n"
    "Use tools to solve tasks."
)
```

#### 2. 工具 description - 环境适配示例

```python
TOOLS = [
    {"type": "function", "function": {
        "name": "bash",
        "description": "Run shell commands. Prefer read_file/glob for file operations. "
                       "Windows: use dir, type, not ls, cat.",
    }},
    {"type": "function", "function": {
        "name": "grep",
        "description": "Fast content search. Use for finding patterns in codebases. "
                       "Windows: works in git bash / WSL.",
    }},
    {"type": "function", "function": {
        "name": "glob",
        "description": "Find files by name patterns. Recommended for file discovery.",
    }},
    # ... 其他工具类似
]
```

## 核心原则

1. **不教具体命令差异** - 只引导 LLM 选择更稳定的工具
2. **工具 description 是第一道防线** - 环境适配细节写在工具定义里
3. **SYSTEM prompt 是总策略** - 给出工具优先级和使用场景
4. **根据工具数量调整** - 工具少时方案3够用，工具多时用本思路
